# Выбор системы мониторинга (LADR)

## Контекст
Нам необходимо развернуть систему мониторинга для отслеживания состояния серверов и приложений. Требования:
- Сбор метрик с серверов (CPU, RAM, диск, сеть)
- Сбор метрик из приложений (HTTP-запросы, ошибки, latency)
- Визуализация данных через дашборды
- Возможность масштабирования
- Бесплатное решение с открытым кодом

## Решение
Выбран стек **Prometheus + Grafana** по следующим причинам:
- **Prometheus** - стандарт де-факто для сбора метрик в современных инфраструктурах
- **Grafana** - мощный инструмент визуализации с поддержкой Prometheus как источника данных
- Наше приложение на Flask может отдавать метрики в формате Prometheus через библиотеку `prometheus_flask_exporter`
- Широкая поддержка экспортеров (Node Exporter для метрик сервера)

## Последствия
- Потребуются дополнительные ресурсы для хранения метрик
- Необходимо настроить сбор метрик с каждого сервера и приложения
- Команда получит наглядные дашборды для мониторинга состояния системы
